# TELEGRAM-БОТ. Бот-квиз/подбор для бизнеса

## Цель бота
Показать логику ветвления: несколько вопросов → рекомендация продукта/тарифа/услуги → сбор контакта. Пример ниши: подбор тарифа для фитнес-клуба (можно заменить на любую — курс, продукт, услугу).

## Схема диалога

```
/start
  → «Привет! Отвечу за 1 минуту, какой формат тренировок подойдёт именно вам. Начнём?»
  → Кнопка [Начать подбор]

Вопрос 1: «Как часто планируете заниматься?»
  Кнопки: 1-2 раза в неделю / 3-4 раза в неделю / Каждый день

Вопрос 2: «Что вам важнее?»
  Кнопки: Похудение / Набор массы / Общий тонус и здоровье

Вопрос 3: «Предпочитаете тренироваться...»
  Кнопки: Самостоятельно / С тренером / В группе

Вопрос 4: «Какой у вас опыт?»
  Кнопки: Новичок / Средний уровень / Продвинутый

--- Логика ветвления ---
На основе комбинации ответов бот выдаёт одну из 3-4 заготовленных рекомендаций:
  - Если «1-2 раза» + «Самостоятельно» + «Новичок» → тариф «Старт» (базовый абонемент + вводная консультация)
  - Если «3-4 раза» + «С тренером» → тариф «Персональный» (индивидуальные тренировки)
  - Если «Каждый день» + «Продвинутый» → тариф «Профи» (безлимит + доп.услуги)
  - Дефолтный вариант для прочих комбинаций → тариф «Стандарт»

Результат:
  → «Судя по ответам, вам подойдёт тариф [Название]. Он включает: [краткое описание]. Стоимость: от [цена] ₽/мес»
  → Кнопка [Узнать подробнее] [Оставить заявку на консультацию]

[Оставить заявку] →
  Запрос имени → запрос телефона → подтверждение
  → Клиенту: «Спасибо! Мы свяжемся с вами в течение часа»
  → Админу: уведомление с результатами квиза и контактом клиента
```

## Технические детали
- Реализация: последовательность инлайн-кнопок, ответы сохраняются в переменные сессии пользователя (state machine — важно продумать хранение состояния диалога для каждого пользователя)
- Логика подбора — простая матрица условий (if/elif по комбинации ответов) или система баллов (каждый ответ даёт баллы разным тарифам, побеждает тариф с наибольшим количеством баллов — более гибкий вариант, лучше смотрится в портфолио)
- Данные о прохождении квиза + контакт — сохранять в таблицу для последующей выгрузки

## Демо-данные для портфолио
- Ниша: фитнес-клуб (можно переименовать под любой другой бизнес — курсы, консалтинг, услуги)
- 4 вопроса, 3-4 варианта результата
- Понятные названия тарифов с ценами

## Подача в портфолио
Запишите короткий скринкаст (10-15 сек, экран телефона) — GIF или видео прохождения квиза целиком, от старта до выдачи рекомендации.
